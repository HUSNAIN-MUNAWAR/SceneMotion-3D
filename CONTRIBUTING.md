# Contributing

## Setup

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r backend/requirements.txt
make test
```

## Branch naming

Use `feature/...`, `fix/...`, `docs/...`, or `experiment/...`.

## Tests

Run:

```bash
make test
python -m compileall backend vision_core workers scripts
```

## Code style

Prefer typed, small modules. Keep failure cases explicit. Do not hide exceptions behind fake success messages.

## Adding a feature pipeline

Add config under `configs/pipeline/`, implementation under `vision_core/features/` or `vision_core/tracking/`, tests under `tests/`, and docs explaining limitations.

## Adding a depth provider

Register it in `vision_core/depth/provider_registry.py`. Do not auto-download huge weights by default. Add provider metadata explaining whether the output is metric.

## Adding a dataset adapter

Add an adapter under `vision_core/datasets/`, a preparation script under `scripts/`, validation checks, and documentation. Do not bundle large datasets.

## Adding a benchmark

Metrics must be generated by code. Do not add fake benchmark tables.

## Documentation rules

Every README claim must map to code, tests, scripts, artifacts, or a clearly marked future-work item.
